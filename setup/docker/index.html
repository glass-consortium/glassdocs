Setup Docker Image

Download and setup docker image: glasstools/keystone

Looking for Singularity image?

Visit Singularity page for installation and configuring Singularity based workflows.

Installing Docker

Your system administrator can install Docker version 17.09.0-ce, build afdb6d4 or higher. For more, visit https://docs.docker.com/engine/installation/

Docker image

docker pull glasstools/keystone
# OR
docker pull glasstools/keystone:1.2.2

https://hub.docker.com/r/glasstools/keystone

Test run

With data mounts and host user:group mapping

export GLASSMOUNTS="/fastscratch/foo/dockvol/glass_tools/glass_mounts"

docker run -e HOSTUSER=$USER -e HOSTGROUP=verhaak-lab -e HOSTUSERID=$UID -e HOSTGROUPID=56565 -v "${GLASSMOUNTS}"/disk1/glasscore:/mnt/glasscore -v "${GLASSMOUNTS}"/disk1/scratch:/mnt/scratch -v "${GLASSMOUNTS}"/disk2/glassdata:/mnt/glassdata glasstools/keystone
  • This will source commands or workflow from disk1/glasscore/Workflows/startup script
  • Visit Disk Mounts section for more on how to mount host disk volumes and start workflows.

Run specific tools

To run specific command (user id mapped to host user), use "-c bash command" format immediately after docker run ... glasstools/keystone.

export GLASSMOUNTS="/fastscratch/foo/dockvol/glass_tools/glass_mounts"

docker run -e HOSTUSER=$USER -e HOSTGROUP=verhaak-lab -e HOSTUSERID=$UID -e HOSTGROUPID=56565 -v "${GLASSMOUNTS}"/disk1/glasscore:/mnt/glasscore -v "${GLASSMOUNTS}"/disk1/scratch:/mnt/scratch -v "${GLASSMOUNTS}"/disk2/glassdata:/mnt/glassdata glasstools/keystone "-c samtools"

## view data mounts
docker run -e HOSTUSER=$USER -e HOSTGROUP=verhaak-lab -e HOSTUSERID=$UID -e HOSTGROUPID=56565 -v "${GLASSMOUNTS}"/disk1/glasscore:/mnt/glasscore -v "${GLASSMOUNTS}"/disk1/scratch:/mnt/scratch -v "${GLASSMOUNTS}"/disk2/glassdata:/mnt/glassdata glasstools/keystone "-c ls -alh /mnt/"

Without data mounts

docker run glasstools/keystone

keystone image is configured to be run with structured data mounts as shown under glass_mounts directory. Barebone docker run glasstools/keystone will quit immediately with partial message like following:

User and Group ID mapping: ENABLE Running userid_mapping.sh script

disable usermapping

PS: Without host user:group mapping, docker container will run under default glassuser which has limited read:write permissions to host file system.

Supply -e USERMAPPING=DISABLE while starting docker container to disable user mapping to host system.

Unlike user mapped, data mounts above, note change in command format when you disable user mapping, i.e., supply bash command in quoutes without prefix -c

docker run -e USERMAPPING=DISABLE -v "${GLASSMOUNTS}"/disk1/glasscore:/mnt/glasscore -v "${GLASSMOUNTS}"/disk1/scratch:/mnt/scratch -v "${GLASSMOUNTS}"/disk2/glassdata:/mnt/glassdata glasstools/keystone "id -a && ls -alh /mnt/"
  • Note that docker container running as a glassuser has not valid permission to access mounted host volumes.
